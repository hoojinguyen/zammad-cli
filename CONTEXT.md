# Zammad CLI Installer Context

A modular tool for automated installation and command-line management of Zammad Helpdesk across multi-platform host systems.

## Language

**Bootstrap Script**:
A lightweight, zero-dependency entry script (`install.sh`) that installs Python 3, Docker, and other initial system prerequisites on the Target OS, and tunes kernel parameters.
_Avoid_: Entrypoint script, setup script

**Zammad CLI**:
The Python-based command-line interface tool (`zammad-cli`) implemented using Python 3's standard library, used to configure, deploy, run, and manage Zammad and its services.
_Avoid_: Manage script, control utility

**Target OS**:
The operating system of the host machine (specifically supporting Ubuntu and macOS) where Zammad will be installed and run.
_Avoid_: Server OS, platform

**Installation Log**:
The persistent log file (`zammad-install.log`) stored in the installation directory that captures the detailed stdout and stderr of the installation and management process.
_Avoid_: System log, debug output

**Container Stack**:
The set of Docker containers and volumes running Zammad, including the Ruby on Rails web/scheduler services, Nginx reverse proxy, PostgreSQL database, Redis, Elasticsearch, and Memcached.
_Avoid_: Application stack, services

**Interactive Mode**:
The text-based menu dashboard displayed when the user executes the Zammad CLI without subcommands, allowing interactive selection of management operations.
_Avoid_: GUI, TUI console

**Database Connection Guide**:
The printable instructions generated by Zammad CLI (`zammad db info`) showing database credentials, host port bindings, and connection instructions for host clients and graphical tools.
_Avoid_: DB config, connection details

**Volume-based Backup**:
A full-stack gzip tarball archive generated by Zammad CLI (`zammad backup create`) containing compressed data from all Zammad persistent Docker volumes (PostgreSQL data, Elasticsearch data, Redis data, and Zammad storage).
_Avoid_: Stack dump, file copy
